What the Forge Does

Yangzhi Audiobook Forge is a native Windows front end for an existing Chatterbox Turbo portable installation. It sends your text and reference voice to the local engine, divides long writing into structure-aware chunks, joins those chunks with optional crossfades, and saves a descriptively named audiobook WAV.

Before You Begin

You need

  • 64-bit Windows 10 or 11
  • A working Chatterbox Turbo portable folder
  • A clear reference voice in WAV format
  • A UTF-8 poem, story, or narration in TXT format
  • An existing output folder
  • Free space for uncompressed WAV audio

You do not need separately

  • Python installed system-wide
  • Tkinter or a GUI BAT launcher
  • Visual Studio
  • FFmpeg for the Forge joining workflow

The necessary AI dependencies must already exist inside the selected portable Chatterbox environment.

Expected Turbo folder

Chatterbox Turbo\ ├── python_embedded\ │ └── python.exe ├── cbx_turbo_cli.py ├── reference_audio\ └── model and cache files
Best reference audio: use one speaker, clean natural speech, little room echo, and no music or effects. A focused 10–30 second sample is a dependable starting point.

First-Run Setup

  1. Extract the ZIP. Keep the executable anywhere convenient. Do not run it directly from inside the ZIP preview.
  2. Open the app. If SmartScreen appears, confirm that the file came from the official project page, choose More info, then Run anyway.
  3. Choose Turbo folder. Select the root folder containing python_embedded\python.exe and cbx_turbo_cli.py.
  4. Choose Text / poem. Select a UTF-8 .txt file. Plain text gives the most predictable result.
  5. Choose Reference voice. Select the WAV recording of the voice you own or have permission to use.
  6. Choose Output folder. Select an existing folder where the finished WAV will be saved.
  7. Forge. Pick a performance style, check the values, and click FORGE AUDIOBOOK VOICE.

The app remembers its fields in %APPDATA%\Yangzhi Audiobook Forge\settings.ini.

Control Reference

ControlPurposePractical advice
TemperatureControls variation and performance unpredictability.Lower values are steadier and usually preserve voice identity better; higher values can be expressive but may distort pronunciation or accent. Accepted range: 0.35–1.20.
SpeedApplies the final playback speed factor.1.00 is unchanged; 1.06 is 6% faster. Test voice identity at 1.00 first. Accepted range: 0.50–2.00.
ExaggerationInfluences emotional expressiveness.Start near 0.50. Accepted range: 0–2, or leave blank if your CLI supports omission.
CFG weightBalances guidance and naturalness in supported Chatterbox workflows.Turbo may ignore this control. Start near 0.50, or leave it blank.
SeedControls the random choices used to create a take.The default is 42. The custom CLI applies the same fixed seed to every chunk and logs it. Change the seed for a different reproducible take; leave it blank for uncontrolled randomness.
Chunk charsMaximum approximate text size sent per segment.200 is the built-in default. For audiobook tests, 225–240 may reduce the number of chunk openings while preserving sentence boundaries. Range: 40–1000.
Crossfade msOverlaps the end of one segment with the start of the next to soften clicks or hard seams.Long fades may mask opening syllables. Use 0 for diagnosis; 5–12 ms is a conservative audiobook range. It cannot repeat an entire line. Range: 0–1000 ms.
Lead-in msAdds silence before the completed narration begins.This protects the episode opening from player clipping but does not alter later chunks or repair speech already generated incorrectly. 250 ms is a practical test value. Range: 0–10000 ms.

Performance Styles

The style menu provides quick temperature starting points. After choosing a style, all visible advanced fields remain editable.

StyleTemperatureGood starting use
Natural / Neutral0.78General narration and balanced prose
Calm / Instructor0.62Lessons, guides, reflective writing
Warm / Friendly0.70Conversational stories and welcoming narration
Storyteller0.88Fiction, folktales, and colorful prose
Dramatic / Wuxia1.05Poetry, martial epics, and heightened performance
Crisp / Announcer0.55Direct, controlled, promotional delivery
CustomNo automatic changeManual experimentation
A higher temperature is not automatically “better.” For a long work, test one short paragraph first and listen for pronunciation, pacing, consistency, and unwanted dramatic variation.

Turbo Emotion & Reaction Tags

Chatterbox Turbo v2 supports the nine native paralinguistic tags below. Type the exact lowercase tag directly into the TXT narration where the reaction should occur.

TagEffectExample
[laugh]A distinct laugh.At last, the treasure is mine! [laugh]
[chuckle]A smaller, gentler amused reaction.You thought the old master was sleeping. [chuckle]
[sigh]An audible sigh.Another mountain staircase. [sigh]
[gasp]A sudden intake of breath.The moon gate opened. [gasp]
[cough]A cough.I meant to do that. [cough]
[clear throat]A throat-clearing sound.[clear throat] Disciples, listen carefully.
[sniff]A sniff.It is only mountain rain. [sniff]
[groan]A groan of effort or frustration.The manuscript has revisions. [groan]
[shush]A quieting “shh” sound.[shush] Something is moving outside.
Use tags sparingly. Put a space around a tag, keep it close to the phrase it belongs to, and test a short passage before generating a whole chapter. The custom CLI preserves poem and stanza line breaks, keeps tags with neighboring speech at chunk boundaries, and prints this supported-tag list in the Generation Log.
Important: mood directions such as [sad], [angry], and [whisper] are not native tags in this workflow. For mood, shape the wording and punctuation, choose a performance style, and adjust Exaggeration and Temperature. Results still vary with the reference voice, seed, settings, and surrounding text.

Recommended Audiobook Workflow

  1. Clean the reference WAV and input text.
  2. Generate one short test passage with Natural / Neutral, Temperature around 0.70–0.78, Speed 1.00, and Seed 42.
  3. Adjust speed and performance controls in small steps.
  4. Keep the seed fixed while comparing settings.
  5. Use a different seed when you want a new take for garbled, repeated, or poorly pronounced speech.
  6. When satisfied, generate the complete text.
  7. Watch the live log; it prints the fixed seed used for each chunk. Do not close the app during generation.
  8. Use Open Output and audition the title, chunk openings, tag-adjacent speech, transitions, and ending with headphones.
  9. Keep the original text, voice sample, settings, and final WAV together as a project archive.

Output names record the complete test identity: [text filename][reference WAV][style][speed][seed].wav. Example: [Episode One][Paul neutral][Natural - Neutral][1.00][42].wav. Reusing every field produces the same filename, so move or rename a take first if you want to preserve multiple runs with identical settings.

Voice consent: clone only your own voice or a voice for which you have clear permission. Do not use synthetic speech for deceptive impersonation, fraud, harassment, or bypassing identity checks.

Troubleshooting

ProblemWhat to check
Embedded Python not foundYou selected the wrong folder level. The chosen root must directly contain python_embedded.
Turbo CLI not foundConfirm the latest cbx_turbo_cli.py exists directly inside the chosen Turbo root.
Output folder does not existCreate the folder first, then select it again.
Generation ends with an errorRead the final lines in the Generation Log. First verify that Chatterbox Turbo works independently on the same computer.
Voice does not resemble the reference or has an odd accentUse a clean natural reference, select Natural / Neutral, lower Temperature to roughly 0.70–0.78, set Speed to 1.00, and test another fixed seed.
Opening words are garbledWrite titles naturally, such as Episode One. The Invitation. Lead-in protects playback but cannot repair speech Turbo generated incorrectly. Try another seed.
Words near reaction tags are garbledGive tags punctuation and breathing room. Avoid attaching a tag directly to words, stacking tags, or placing one in the middle of a sentence.
A line is repeatedCrossfade cannot repeat a whole line. Test another fixed seed and inspect whether repetition is generated within a chunk.
Opening syllables at joins sound maskedReduce Crossfade; use 0 ms diagnostically, then try 5–12 ms if smoothing is needed.
Joins click or sound abruptTry a small crossfade, sentence-friendly chunk size, and cleaner punctuation.
Generation is slowCPU-only generation can be slow. Confirm that the selected Turbo installation is configured for the intended GPU/PyTorch build.
Settings need a full resetClose the app and remove %APPDATA%\Yangzhi Audiobook Forge\settings.ini.
