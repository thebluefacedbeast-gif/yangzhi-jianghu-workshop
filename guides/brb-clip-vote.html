BRB Clip Vote System

Install the interactive OBS overlay that lets chat vote for one of three Kick clips, then plays the winner with audio.

Watch the YouTube demonstration ▶

Streamlined OBS + Streamer.bot Setup Guide

Simple install notes for the working Kick-direct clip vote system, with optional local test mode.

1. Before You Use This: Change These Settings

Where Change this Notes
Streamer.bot → BRB Vote - Pick Kick Clips targetUserName = YOUR_KICK_CHANNEL_NAME This is the main channel-specific setting. It controls which Kick channel clips are pulled from.
OBS Scene name = BRB Clip Vote
Browser source = BRB_ClipVote_Overlay		 Keep these names unless you also update the Streamer.bot actions and overlay references.
Point Browser source to = brb_clip_vote.html file.
HTML CONFIG block TITLE_TEXT / SUBTITLE_TEXT / TIMER_LABEL_TEXT Optional branding text. Leave STREAMERBOT_WS_URL as ws://127.0.0.1:8080/ unless your Streamer.bot WebSocket port is different.
Optional local mode BRB Vote - Pick Local Clips → clipFolder Only needed for local mode. Also run the local clip server BAT.

2. What This System Does

  • When the BRB Clip Vote scene starts, Streamer.bot pulls Kick clips directly, picks three random options, and sends those playable URLs to the HTML overlay.

  • Chat votes by typing 1, 2, or 3. Vote totals update live under each panel.

  • When the timer reaches zero, voting locks and the winning clip expands and plays with audio.

  • The overlay waits for the winning video to finish, then calls BRB Vote - Winner Finished. The system starts the next round only if the BRB scene is still active.

  • If nobody votes, the system picks a random winner. Ties are broken randomly among tied clips.

3. Required Names

Item Required / Recommended Name Notes
OBS scene BRB Clip Vote Controller uses this scene name to start/stop the system.
Browser source BRB_ClipVote_Overlay Loads the voting overlay. 1920 x 1080 recommended.
Overlay file brb_clip_vote.html Use the clean unified HTML file.
Background source Vote Background Any background image/video/layout source.

4. OBS Setup

1. Create a scene named BRB Clip Vote.

2. Add your background source first.

3. Add a Browser Source named BRB_ClipVote_Overlay.

4. Set the Browser Source size to 1920 x 1080.

5. Load brb_clip_vote.html as the browser source file.

6. Put the browser source above the background source.

7. For winner audio, make sure the browser source is not muted. If using “Control audio via OBS,” check the OBS mixer and Advanced Audio Properties.

5. Streamer.bot WebSocket Setup

  • Open Streamer.bot before testing the overlay.

  • Make sure Streamer.bot is connected to OBS.

  • Enable Streamer.bot WebSocket Server.

  • Default overlay setting: ws://127.0.0.1:8080/.

  • Authentication should remain off unless you also update the HTML connection logic.

  • The overlay reads Non-Persisted Globals through WebSocket polling.

6. Streamer.bot Actions

Action Simple job
BRB Vote - Controller Starts Initialize when entering BRB Clip Vote; runs Stop when leaving.
BRB Vote - Initialize Enables the loop, resets state, sets loading, then calls Start Round.
BRB Vote - Start Round Resets votes/timer, picks three clips, opens voting, starts countdown.
BRB Vote - Pick Kick Clips Pulls Kick clips, picks three candidates, resolves playable URLs.
ClipCandidate1 / 2 / 3 Passes each chosen clip ID into Resolve Kick Clip Slot.
Resolve Kick Clip Slot Runs Kick → GetClipVideoUrl, then stores the resolved URL.
Store Resolved BRB Vote Clip Writes brbVoteClip1/2/3 and optional title/preview/page globals.
BRB Vote - Pick Local Clips Optional local test picker. Scans a local folder and sets filename globals.
BRB Vote - Chat Vote Accepts chat messages 1, 2, or 3 and tracks one vote per viewer.
BRB Vote - End Round Locks voting, handles ties/no votes, sets brbVoteWinner and status winner.
BRB Vote - Play Winner No fixed delay. The overlay plays the winner and calls Winner Finished when video ends.
BRB Vote - Winner Finished Safety bridge from overlay. Calls Next Round only if status is winner and continue is true.
BRB Vote - Next Round Starts another round if brbVoteContinue is true.
BRB Vote - Stop Stops loop, clears votes/winner/clips/timer, returns status to idle.

7. Changing the Voting Time

To increase or decrease the voting window, change the brbVoteTimer value in these Streamer.bot actions:

Action What to change
BRB Vote - Initialize Set brbVoteTimer to the desired default, such as 30, 45, or 60.
BRB Vote - Start Round Set brbVoteTimer to the same value before countdown_timer runs. This controls each live round.
BRB Vote - Stop Set brbVoteTimer to the same value so the overlay resets cleanly when leaving the scene.

Example: For a 45-second vote, set brbVoteTimer = 45 in Initialize, Start Round, and Stop.

9. Clip Source Modes

Mode Streamer.bot action to call in Start Round HTML behavior
Online Kick mode BRB Vote - Pick Kick Clips Uses direct https:// video URLs. No local server needed.
Optional local mode BRB Vote - Pick Local Clips Receives filenames and routes them through LOCAL_CLIP_SERVER, usually http://127.0.0.1:8123/. Run the local server BAT first.

10. Non-Persisted Globals

Global Purpose
brbVoteStatus idle, loading, voting, or winner.
brbVoteMode Preview mode: live or thumbnail.
brbVoteTimer Countdown time in seconds.
brbVote1 / brbVote2 / brbVote3 Vote totals.
brbVoteClip1 / brbVoteClip2 / brbVoteClip3 Selected clip URLs or local filenames for current round.
brbVoteVoters Tracks who already voted this round.
brbVoteLocked Prevents new votes after voting closes.
brbVoteContinue Controls whether rounds keep looping.
brbVoteWinner Winning clip number: 1, 2, or 3.

11. First Test Pass

1. Start Streamer.bot and OBS.

2. Switch to BRB Clip Vote.

3. Confirm the overlay appears and clips load.

4. Send chat messages 1, 2, and 3.

5. Confirm vote counts update correctly.

6. Let the timer reach zero.

7. Confirm the winner expands and plays with audio.

8. Wait for the winner clip to finish; confirm the next round starts automatically.

9. Switch away from BRB Clip Vote during voting and during winner playback; confirm the system stops cleanly.
